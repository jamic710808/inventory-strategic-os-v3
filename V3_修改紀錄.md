## v3.0.3 — 2026-03-15（全局篩選器）

### 新增功能

#### 全局篩選列（Global Filter Bar）

- **位置**：所有頁籤內容區頂端，固定顯示（設定 DAX 頁除外）
- **年度篩選**：複選下拉，自動從資料抽取唯一年份，支援多年度並選
- **品類篩選**：複選下拉，自動從資料抽取唯一品類，支援跨品類並選
- **品名/品號搜尋**：模糊比對搜尋框，300ms 防抖，同時比對 ProductName 與 ProductID
- **清除篩選按鈕**：有作用中的篩選條件時才顯示
- **篩選狀態徽章**：顯示「已篩選 N 項條件」

#### 篩選架構設計

- 篩選在 `rawData` 原始資料層執行（非顯示層），確保 KPI、月度聚合、季節指數、報告文字全部同步更新
- 新增 `_rawDataFull` 變數永久持有完整未過濾資料集
- 新增 `_skipRawUpdate` 旗標，讓 `processData()` 可被篩選重用而不覆蓋原始資料
- 篩選後執行 `processData(filtered)` 重新聚合，所有 render 函式自動讀取篩選後結果
- `clearGlobalFilter()` 還原至完整原始資料
- 匯出功能遵循「所見即所出」原則，跟隨篩選狀態

#### 影響頁籤

全部 13 個資料頁籤（設定 DAX 頁為靜態內容，不適用）：
總覽、趨勢周轉、結構ABC、庫齡風險、預測補貨、供應鏈、品項剖析、策略行動、What-If情境模擬、EOQ成本、FIFO批次、資料品質、報告中心

## v3.0.2 — 2026-03-15（供需熱力圖最適化）

### 修正

#### 供應鏈頁 — 供需熱力圖（品類 × 月份）

**問題**：
- Canvas 尺寸固定（380px），不隨容器縮放
- 僅顯示最後 6 個月（`mKeys.slice(-6)`），遺漏歷史資料
- 沒有 DPR（devicePixelRatio）處理，Retina 螢幕文字模糊
- Y 軸品類標籤截為 4 字，中文品類名顯示不完整
- 數值文字固定白色，淺色格子上難以辨讀

**修正內容**：
- `cellW` 改為動態計算：`Math.max(32, (containerW - padL - padR) / months.length)`，自動填滿容器寬度
- 改為顯示全部可用月份（`mKeys` 完整陣列）
- 加入 DPR 感知：`canvas.width = logicalW * dpr`，`ctx.scale(dpr, dpr)`，`canvas.style.width = logicalW + 'px'`
- `padL` 從 70 調整為 90，`cellH` 從 28 調整為 30
- Y 軸標籤截取長度從 4 字放寬至 5 字
- 數值文字改為自適應色彩：`ratio > 0.45` 時白色，其餘依深/淺色模式選擇深色文字
- 大數值自動縮寫（≥10000 顯示為 `Nk`）
- `canvas.style.maxWidth = '100%'`，`canvas.style.display = 'block'` 防止 overflow

## v3.0.1 — 2026-03-15（JavaScript 語法修正）

### 修正

#### V3 初版組裝後 JavaScript 語法錯誤

**根本原因**：
第一次字串修復器（`fix_string_newlines`）遇到 JavaScript 正則表達式字面量 `/"/g` 時，將其中的 `"` 誤判為雙引號字串的開始，導致後續 ~1780 行程式碼被當作字串內部處理，所有程式碼換行符被替換成文字 `\n`，造成整個函式體壓縮成單行導致語法錯誤。

**修正策略**：
1. 撰寫 `reassemble_v3.py`，從原始部件（`v3_parts/part_html.txt`、`v3_parts/part_js.txt`）重新組裝
2. 新版字串修復器加入**正則字面量辨識**邏輯：當 `/` 前方為運算子或關鍵字時（`is_regex_start()` 函式判斷），跳過整個 `/pattern/flags` 區塊，不進行字串處理
3. 修復 37 個字串字面量內的實體換行符（替換為 `\n` 跳脫序列）
4. 使用 `node --check` 驗證語法

**新增工具檔**：
- `reassemble_v3.py`：清潔的重組腳本，包含正則感知字串修復器，可重複執行

### 新增頁籤（V2 → V3）

| 頁籤 | 說明 |
|------|------|
| 預測補貨 | Fisher 季節性檢定、需求預測、95% 信賴區間 |
| What-If 情境模擬 | 4 參數滑桿、悲觀/基準/樂觀三情境比較 |
| EOQ 成本 | EOQ 成本曲線、最佳批量計算、Top 15 分析表 |
| FIFO 批次 | 先進先出批次消耗追蹤、加權平均庫齡 |
| 資料品質 | 6 項自動品質檢查、品質評分 0-100 |
| 報告中心 | 自動週報生成、PDF 匯出、歷史快照管理 |

### 核心演算法

| 演算法 | 說明 |
|--------|------|
| ABC 分類 | 相對法（累積價值比率）或絕對法，A≤50%、B≤70%、C其餘 |
| XYZ 分類 | 變異係數（CoV）法，X<0.5、Y<1.0、Z其餘 |
| FIFO 消耗 | 按入庫批次日期排序，先消耗最舊批次，計算加權平均庫齡 |
| ROP 計算 | ROP = 日均需求 × 前置時間 + 安全庫存；安全庫存 = Z × σ × √前置時間 |
| EOQ 計算 | EOQ = √(2DS/H)，D=年需求、S=訂購成本、H=持有成本 |
| Fisher 季節性 | 月間變異 ÷ 年間變異比值，>6.5 判定具顯著季節性 |
| 雙軌 STR | 快銷品（出庫÷入庫）/ 慢銷品（出庫÷期初庫存） |
| 雙軌 SSR | 長週期（庫存÷銷售額）/ 短週期（期初+入÷出） |

### 全局特性

- **零依賴**：所有功能內嵌於單一 HTML 檔案，CDN 函式庫載入後可離線使用
- **懶載入**：切換頁籤時才呼叫對應 `render*()` 函式，避免首次載入過長
- **深/淺色模式**：CSS 變數驅動，localStorage 記憶偏好
- **響應式**：支援桌面、平板（最大寬度 1600px）
- **DPR 感知**：Canvas 繪圖考慮 devicePixelRatio，Retina 螢幕清晰顯示
- **自動載入 Demo**：`DOMContentLoaded` 事件觸發 `generateDemoData()`，開啟即有範例資料
