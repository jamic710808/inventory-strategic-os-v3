**新增模組：經濟訂購量與三成本分析**

| **訂購頻率分析** | 目前 vs 建議的補貨週期對比 |

**EOQ 公式：**
```
EOQ = √(2 × D × S / H)

D = 年需求量
S = 每次訂購固定成本
H = 每單位每年持有成本 = 單位成本 × 持有成本率

預設參數（可配置）：
  S = 500 元/次
  持有成本率 = 25%/年
```

### 🆕 Tab 13：FIFO 批次追蹤

**技術報告來源：** 第四章 4.2-4.3 節「FIFO 倒推路徑」

| 元素 | 說明 |
|------|------|
| **品項選擇器** | 下拉選單選擇品項 → 顯示該品項的批次 FIFO 堆疊圖 |
| **批次消耗瀑布圖** | 每批入庫量 → 被出庫消耗量 → 剩餘量的瀑布式展示 |
| **庫齡天數分佈** | 以氣泡圖呈現各批次的剩餘量（氣泡大小）× 庫齡（X軸）× 成本（Y軸） |
| **FIFO 動畫模擬** | 按下「播放」按鈕，逐日模擬 FIFO 消耗過程 |

**FIFO 核心算法（技術報告 4.3.2 節轉譯為 JavaScript）：**

```javascript
function calcFIFO(batches, totalOut) {
    // batches = [{date, qty, cost}, ...] 按日期排序
    // totalOut = 截至統計日的總出庫量
    let cumIn = 0;
    return batches.map(b => {
        cumIn += b.qty;
        if (cumIn <= totalOut) {
            return { ...b, remaining: 0 }; // 全部消耗
        }
        const remaining = Math.min(b.qty, cumIn - totalOut);
        return { ...b, remaining };
    }).filter(b => b.remaining > 0);
}
```

```javascript
// V3 新增計算欄位
p.xyz = coV < 0.5 ? 'X' : (coV < 1.0 ? 'Y' : 'Z'); // XYZ 分類
p.eoq = Math.sqrt(2 * p.annualDemand * orderCost / holdingCost);
p.fifoAging = calcFIFO(p.batches, p.totalOut); // FIFO 庫齡
p.seasonalIndex = monthAvg / yearAvg; // 各月季節指數
p.forecastComposite = trendBase * seasonalIndex; // 複合預測
p.strType = p.leadTime > 21 ? 'slow' : 'fast'; // 售磬率口徑
```

## 七、與 V2 的差異化定位

```
V1：基礎庫存看板（8頁、8圖表、基本 ABC/ROP）
     ↓ 每個模組都更深入
V2：進階分析平台（9頁、18+圖表、GMROI/Fisher/背離偵測）
     ↓ 從「被動分析」進化為「主動決策」
V3：決策支援系統（14頁、30+圖表、FIFO/EOQ/情境模擬/品質控管）
```

**V3 的核心定位：不只是看數據，更是做決策。**

每個分析模組都連結到「下一步行動」：
- 看到庫齡惡化 → 跳到行動清單 → 標記處理進度
- 預測庫存不足 → What-If 模擬不同補貨策略 → 匯出補貨建議報告
- EOQ 偏離 → 調整訂購參數 → 即時看到成本影響
