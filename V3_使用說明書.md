## 1. 系統概述

庫存分析智能儀表板 V3.0 是零依賴單一 HTML 檔案的互動式庫存分析工具，直接用瀏覽器開啟即可使用，無需安裝任何軟體。

### 技術架構

| 元件 | 版本 / 來源 |
|------|------------|
| Chart.js | 4.4.0（UMD，CDN） |
| chartjs-plugin-datalabels | 2.2.0（CDN） |
| SheetJS（xlsx） | 0.20.3（cdn.sheetjs.com） |
| Lucide Icons | latest（unpkg） |
| html2canvas | 1.4.1（CDN） |
| jsPDF | 2.5.1（CDN） |
| Tailwind CSS | CDN（utility 樣式輔助） |

> **注意**：首次開啟需要網路連線以載入 CDN 函式庫。離線環境請事先開啟一次使瀏覽器快取。

### 核心指標（KPI）

| 指標 | 說明 |
|------|------|
| 庫存總金額 | 所有品項現有庫存 × 單位成本 |
| DSI（天） | 庫存天數 = 庫存金額 ÷ 日均銷售成本 |
| 存貨周轉率 | 年化周轉次數 = 年化 COGS ÷ 平均庫存 |
| 庫存健康分 | 0–100 分，由 DSI / 庫齡 / 缺貨 / GMROI 加權計算 |
| GMROI | 毛利庫存投資報酬率 = 毛利 ÷ 平均庫存成本 |
| 售罄率（STR） | 出庫量 ÷ (入庫量 or 期初庫存)，快/慢銷品雙軌 |
| 缺貨品項數 | 現有庫存低於 ROP 的品項數量 |
| >90天庫齡佔比 | 高風險庫存佔總庫存金額比例 |

## 2. 快速開始

### 步驟一：開啟儀表板

直接雙擊 `Inventory_Strategic_OS_V3.html`，或在瀏覽器中開啟（建議 Chrome / Edge 最新版）。

開啟後系統自動載入 **Demo 示範資料**（50 個品項 × 180 天 = 9,000 筆記錄），可立即瀏覽所有頁籤。

### 步驟二：上傳自有資料

點選右上角 **「↑ 上載資料」** 按鈕，或切換至「設定 DAX」頁使用拖曳上傳區，選擇符合格式的 Excel 檔案（`.xlsx`）。

成功解析後，儀表板自動重新計算所有指標並更新圖表。

### 步驟三：使用篩選器聚焦分析

在所有頁籤頂部可見**全局篩選列**，可按年度、品類、品項搜尋縮小分析範圍（詳見第 3 節）。

### 步驟四：匯出成果

- 右上角 **「↓ 下載資料」**：匯出 Excel（含 KPI、品項分析、ROP 補貨、策略行動 4 個工作表）
- 報告中心頁：匯出 PDF 報告、複製文字報告、儲存歷史快照

## 3. 全局篩選器

篩選列位於所有頁籤內容區的最頂端（設定 DAX 頁除外），三個維度可單獨或組合使用。

```
┌─────────────────────────────────────────────────────────────────┐
│  [📅 年度：全部 ▼]  [🏷 品類：全部 ▼]  [🔍 品名/品號搜尋…]       │
│  [✕ 清除篩選]  [已篩選 N 項條件]                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 年度篩選

- 點擊「**年度**」按鈕展開下拉，勾選要保留的年份
- 選項從資料自動抽取，多年度資料自動出現多個選項
- 全部取消勾選 = 等同全選

### 品類篩選

- 點擊「**品類**」按鈕展開下拉，勾選要保留的品類
- 支援複選（例如同時選「電子產品」和「家用電器」）
- 全部取消勾選 = 等同全選

### 品項搜尋

- 在搜尋框輸入關鍵字（品名或品號均可），支援模糊比對
- 輸入後 300ms 自動觸發篩選，無需按 Enter
- 大小寫不敏感

### 篩選作用範圍

篩選在**原始資料層**執行，所有頁籤的圖表、KPI、表格均同步更新，包括：

- 總覽 KPI 及月度趨勢
- 趨勢周轉圖表（月份序列依年份切段）
- ABC/XYZ 結構分析
- 庫齡風險分析
- 預測補貨（ROP 表格、季節指數）
- 供應鏈熱力圖及補貨排程
- 品項剖析表格
- 策略行動告警
- What-If 情境模擬（加權平均依篩選結果計算）
- EOQ 成本分析
- FIFO 批次分析
- 資料品質報告
- 報告中心週報文字

### 清除篩選

點擊「**✕ 清除篩選**」按鈕（有作用中的篩選條件時才出現），恢復顯示全部資料。

## 4. Excel 資料格式

### 必要欄位

| 欄位名稱 | 說明 | 範例值 |
|----------|------|--------|
| `ProductID` | 品項代碼（唯一識別） | `P1001` |
| `ProductName` | 品項名稱 | `智慧型手機` |
| `Category` | 品類 | `電子產品` |
| `Date` | 日期（YYYY-MM-DD） | `2025-10-01` |
| `in_qty` | 當日入庫數量 | `50` |
| `out_qty` | 當日出庫數量 | `12` |
| `current_inventory` | 當日庫存數量 | `188` |
| `aging` | 庫齡（天） | `15` |
| `UnitCost` | 單位成本（元） | `800` |
| `UnitPrice` | 單位售價（元） | `1200` |
| `LeadTime` | 前置時間（天） | `21` |

### 欄位別名對應

系統支援多種欄位命名，自動識別常見變體：

| 標準欄位 | 可接受的別名 |
|----------|------------|
| ProductID | `product_id`、`pid`、`品項ID`、`品號` |
| ProductName | `product_name`、`名稱`、`品名` |
| Category | `品類`、`分類` |
| in_qty | `入庫`、`in` |
| out_qty | `出庫`、`out` |
| current_inventory | `庫存`、`inventory` |

### 資料結構範例

第 1 行為欄位標題，第 2 行起為資料：

```
ProductID | ProductName | Category | Date       | in_qty | out_qty | current_inventory | aging | UnitCost | UnitPrice | LeadTime
P1001     | 智慧型手機   | 電子產品  | 2025-10-01 | 50     | 12      | 188               | 15    | 800      | 1200      | 21
P1001     | 智慧型手機   | 電子產品  | 2025-10-02 | 0      | 8       | 180               | 16    | 800      | 1200      | 21
```

> 範例資料檔案：`Sample_Data_V1.xlsx`、`Sample_Data_V2.xlsx`（可下載後修改使用）

## 5. 14 個分析頁籤

### 頁籤 1：總覽（Overview）

- **14 個 KPI 卡片**：庫存金額、DSI、周轉率、健康分、GMROI、STR、缺貨數、庫齡佔比等
- **月度庫存金額 + COGS 趨勢線圖**
- **品類庫存佔比環圈圖**
- **供應鏈摘要面板**：GMROI、缺貨損失估算、平均前置時間

### 頁籤 2：趨勢周轉（Trend）

- **DSI + 周轉率雙軸圖**（柱線混合）
- **售罄率（SSR）**：長週期（出庫÷入庫）/ 短週期（期初+入÷出）可切換
- **月環比成長率柱圖**
- **移動平均線**：MA3（3個月）、MA6（6個月）

### 頁籤 3：結構 ABC（Structure）

- **ABC 庫存佔比圓餅圖**：A 類（前50%價值）/ B 類（50-70%）/ C 類（其餘）
- **Pareto 累積曲線**（Top 20 品項）
- **供需失衡散佈圖**：COGS 佔比 vs 庫存金額佔比
- **ABC-XYZ 9宮格矩陣**：品項數 + 金額熱力顯示
  - ABC 分類法：相對法（比例累計）/ 絕對法 可切換

### 頁籤 4：庫齡風險（Aging）

- **品類 × 庫齡分級堆疊柱圖**（0-30天 / 30-60天 / 60-90天 / 90-120天 / >120天）
- **庫齡損失估算**：可調整各區間損失率（預設：30天 0%、60天 5%、90天 15%、120天 30%、>120天 60%）
- **背離信號偵測**：DSI 偏高 + 出庫量偏低的高風險品項

### 頁籤 5：預測補貨（Forecast）

- **Fisher 季節性檢定**：月間變異 vs 年間變異比值（>6.5 表示具季節性）
- **12 月季節指數雷達圖**
- **歷史 + 預測趨勢線圖**：包含歷史資料、MA3、趨勢線、季節複合預測、95% 信賴區間
- **Top 20 品項 ROP 狀態表**：日均需求、安全庫存、ROP、現有庫存、距 ROP 天數

### 頁籤 6：供應鏈（Supply）

- **品類 GMROI 橫軸柱圖**（各品類加權平均 GMROI）
- **供需熱力圖（品類 × 月份）**：
  - 自動填滿容器寬度（響應式）
  - 顯示全部可用月份
  - Retina/4K 螢幕清晰渲染（DPR 感知）
  - 色彩自適應文字（深色格白字 / 淺色格深字）
- **補貨排程建議表**：依迫切度排序，顯示建議補貨量、補貨時間、預估成本、優先度

### 頁籤 7：品項剖析（Items）

互動式品項明細表，支援多維篩選：

| 篩選器 | 類型 | 說明 |
|--------|------|------|
| 品名/ID 搜尋 | 模糊搜尋 | 即時過濾 |
| 品類 | 下拉單選 | 精確匹配 |
| ABC 級 | 下拉單選 | A / B / C |
| XYZ 級 | 下拉單選 | X / Y / Z |
| 排序 | 下拉選擇 | 庫存金額、庫齡、GMROI、售罄率 |
| 每頁行數 | 下拉選擇 | 10 / 20 / 50 / 100 |

> 注意：品項剖析的本地篩選與全局篩選為 AND 疊加邏輯

### 頁籤 8：策略行動（Actions）

根據設定閾值自動產生告警，分為 7 類：
1. DSI 過高告警
2. 高庫齡品項告警
3. 低 GMROI 告警
4. 缺貨（低於 ROP）告警
5. 供需失衡告警
6. EOQ 偏離告警
7. 資料品質告警

**可調整閾值**：DSI 上限、庫齡上限、GMROI 下限、失衡上限

### 頁籤 9：What-If 情境模擬（Scenario）

四個參數滑桿，即時計算三情境（悲觀/基準/樂觀）對比：

| 參數 | 範圍 | 說明 |
|------|------|------|
| 需求成長率 | -30% ~ +50% | 年度需求變動 |
| 服務水平 | 90% ~ 99% | 影響安全庫存 Z 值 |
| 持有成本率 | 10% ~ 40% | 影響 EOQ 計算 |
| 前置時間調整 | -7 ~ +14 天 | 供應商交期變動 |

比較指標：庫存總額、DSI、安全庫存、持有成本、GMROI

### 頁籤 10：EOQ 成本（EOQ）

- **EOQ 成本曲線圖**：持有成本 + 訂購成本 + 總成本，最低點標示最佳 EOQ
- **Top 15 品項 EOQ 分析表**：年需求量、EOQ、現行批量、偏離率、年度節省成本
- 可調整計算參數：每次訂購成本（元）、年持有成本率（%）

### 頁籤 11：FIFO 批次（FIFO）

- **品項下拉選擇器**：選擇單一品項進行縱深分析
- **批次甘特圖（橫軸堆疊柱）**：每批次的已消耗量 + 剩餘量 + 庫齡
- **批次明細表**：批次日期、進貨量、已耗用、剩餘量、加權平均庫齡、狀態

計算方法：FIFO（先進先出），以最舊批次優先消耗出庫。

### 頁籤 12：資料品質（Quality）

- **品質評分環形圖**（0-100 分）
- **6 項品質檢查清單**：
  1. 日期完整性（是否有缺失日期）
  2. 數量合理性（是否有負值）
  3. 成本完整性（是否有零成本）
  4. SKU 一致性（同 ID 多個品名）
  5. 庫存平衡性（期末 ≠ 期初 + 入庫 - 出庫）
  6. 品類完整性（是否有未分類）
- **異常明細表**（最多 50 筆）

### 頁籤 13：報告中心（Report）

- **自動生成週報文字**：摘要、KPI 表、品類分析、風險提示、改善建議
- **一鍵匯出 PDF**（含系統整體截圖）
- **複製報告**（複製純文字至剪貼板）
- **歷史快照管理**：儲存當前分析結果、列表查閱、刪除

### 頁籤 14：設定 DAX（Settings）

- 拖曳上傳 Excel 資料（可替代右上角按鈕）
- 重置資料（恢復 Demo）
- 匯出 JSON / CSV 格式
- **DAX 公式知識庫**（折疊面板）：DSI、周轉率、GMROI、ABC分類、ROP、EOQ、Fisher 季節性等完整 DAX 公式參考

> 注意：設定 DAX 頁面時，全局篩選列自動隱藏。

在「報告中心」頁點擊「**儲存快照**」，將當前 KPI、品項數據、篩選條件存入 localStorage。

## 9. 常見問題

**Q：上傳後圖表沒有更新？**
A：確認 Excel 檔案包含 `ProductID` 和 `Date` 兩個必要欄位。可先打開瀏覽器開發者工具（F12）的 Console 查看錯誤訊息。

**Q：圖表顯示空白？**
A：確認網路連線正常（Chart.js 等函式庫需從 CDN 載入）。首次開啟需要網路，之後可離線使用。

**Q：篩選後某些頁籤圖表變空？**
A：確認篩選條件下有足夠資料。例如只選「電子產品」品類後，FIFO 頁的品項下拉會只剩電子品項可選。點擊「清除篩選」恢復全部資料。

**Q：PDF 匯出品質不佳？**
A：PDF 匯出使用 html2canvas，受限於瀏覽器渲染。建議使用 Chrome 並在匯出前切換至淺色模式（避免深色背景印刷問題）。

**Q：資料超過 10,000 筆時速度變慢？**
A：系統使用純前端 JavaScript 處理，建議資料量在 50,000 筆以內（50 品項 × 1000 天約 50,000 筆）。超出範圍建議先在 Excel 中按年份分批上傳。
